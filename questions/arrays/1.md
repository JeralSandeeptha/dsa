# 1. Two Sum

Given an array of integers `nums` and an integer target, return indices of the two numbers such that they add up to `target`.

You may assume that each input would have exactly one solution, and you may not use the same element twice.

You can return the answer in any order.

---

**Example 1:**

```bash
Input: nums = [2,7,11,15], target = 9
Output: [0,1]
Explanation: Because nums[0] + nums[1] == 9, we return [0, 1].
```

---

**Example 2:**

```bash
Input: nums = [3,2,4], target = 6
Output: [1,2]
```

---

```bash
Example 3:

Input: nums = [3,3], target = 6
Output: [0,1]
```

---

**Constraints:**

`2 <= nums.length <= 104`
`-109 <= nums[i] <= 109`
`-109 <= target <= 109`

Only one valid answer exists.

<br />

**Follow-up**:

Can you come up with an algorithm that is `less than O(n2) time complexity`?

---

## Thought Process

Question is saying that `Can you come up with an algorithm that is less than O(n2) time complexity?` as a follow up. So if we are not familier with other data structures then only we can proceed with the `O(n2)` time complexity.

- It means we should have to use some kind of nested loops.
- So my solution is get an element, then compared with others until addition is similar to target value.
- If then we return both values indexes.
- Other wise we return a proper message.

- We can track a single element with first loop while we can track others with the nested loop except previous value.

## Solution

```js
const nums = [4, 2, 7, ];
const target = 9;

function twoSum(nums, target) {
    // trying to iterate elements one by one
    // i means the element  
    for (let i = 0; i < nums.length; i++) {
        // 
        // j means the element next to the previous one
        // Previous element is i
        // So next element should be i+1
        for (let j = i+1; j < nums.length; j++) { 
            // if addition is equal to target then return those indexs.
            if (nums[i] + nums[j] === target) { 
                return [i, j];
            };
        };
    };
    return "No Matches Found!";
};

console.log(twoSum(nums, target));
```

## Complexity Analysis

- Most cost operation is nested loop which means `O(n2)`

```cmd
Time complexity: `O(n2)`
```

- I'm not creating any extra array/object that grows with input size.

```cmd
Space complexity: `O(1)`
```

## What I Learnt From This

- With a `For Loop` we can iterate arrays
- We can use a `For Loop` for as a pointer / track one thing.
- If we can use more `For Loops` we can track more things. One loop for one tracking purpose
